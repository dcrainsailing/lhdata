获取特征 get_feature¶

lhdata.features2.get_feature(key='', symbol=None, area: str = None, dt: str = None, day: str = None, start: str = None, end: str = None, fields=None, orderby=None, limit=None, live: bool = False, groupby=None, hk: bool = True, tb=None, dt_col: str = 'datetime', symbol_col: str = 'symbol', engine=None, meta=None, config=None, db=None, db_type: str = 'mysql', **kwargs)[source]¶

对单个特征表进行查询。如：

  • 查询AAPL自2020-07-01的WF特征：

lhdata.get_feature('wf3', 'AAPL US Equity', start='2020-07-01', dt_col='DATE',symbol_col='SYMBOL')

  • 查询000001自2020-01-01的相对hs300的alphav2特征：

lhdata.get_feature('histates_sz.alpha_ch_hs300_v2', '000001 CH Equity', start='2020-01-01', config=23)

Parameters:
  • key – lhdata中的特征关键词，如‘wf3’；或表名，如’histates_sz.alpha_ch_hs300_v2’。如不指定key，则必须同时指定tb和config参数。
  • symbol – 如‘000001 CH Equity’；空(‘’ 或 [] 或 () 或 None) 代表所有股票；支持股票列表或股票元组。
  • area – in [‘ch’, ‘us’, …]。指定symbol区域，不指定则根据symbol（或symbol list的第一个）判定。为空时，自动设置为’ch’。
  • dt – 手动限制日期，如 “> ‘2018-01-01’”，一般用于不含限定日期的查询；指定后会忽略 day、start、end 三个参数。
  • day – 指定日期，等效于设置了start=end=day, 会忽略start、end的输入。
  • start – 起始日期（含），格式如‘2019-01-01’；空(‘’ 或 None) 则自动设置为1992-01-01。
  • end – 终止日期（含），格式如‘2019-09-01’；空(‘’ 或 None) 则自动设置为查询当日。
  • fields – 指定’datetime’、’symbol’以外的字段，默认所有字段。
  • orderby – 如 ‘datetime desc’；多个限制用逗号隔开，如 ‘datetime desc, symbol asc’。
  • limit – 个数限制，可与时间联合使用，获取确定个数数据。
  • live – 当key参数为lhdata中的特征关键词时有效，切换为该特征的盘中表。
  • groupby – 若指定，则返回按该参数group出的dict。
  • hk – 在香港使用需设置为True，lhdata会自动判断。
  • tb – 数据库名+表名，如’histates_sz.alpha_ch_hs300_v2’；或直接输入一个SQLAlchemy.Table实例。
  • dt_col – 表中的时间字段，默认为’datetime’。
  • symbol_col – 表中的股票名字段，默认为’symbol’。
  • engine – SQLAlchemy.engine实例，不指定会尝试从tb中获取或者根据config创建；一般无需指定。
  • meta – SQLAlchemy.meta实例，一般无需指定。
  • config – 数据库连接的ip、账号、密码，格式为dict；或者lhdata中的数据库连接关键词，如‘215’，会使用深圳的215数据库连接。 如不指定，则尝试使用key参数（必须为特征关键词）获取对应的数据库连接，如key参数为’alphav2’。
  • db – 数据库名，如’histates_sz’，一般无需指定，会自动从key参数或者tb中获取。
  • db_type – ‘mysql’ 或 ‘mssql’，默认为’mysql’。
